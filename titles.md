# 标题公式（内置，自给自足）

标题决定打开率，尤其小红书/公众号。这里是一套**内化的通用公式**——不依赖任何外部标题工具。用法：**一次想 3 个不同型号让用户挑**，挑的过程会暴露用户的偏好，记进档案。

## 选型前三条铁律
1. **≤ 20 字**（小红书）；公众号可「打开率标题 + 真实感标题」双标题。
2. **至少含一个钩子要素**：数字 / 痛点 / 反差 / 身份 / 结果 / 悬念，任选其一，别贪多。
3. **标题必须和正文兑现**，别标题党骗点击——一次性买卖，掉粉最快。
4. 语气仍走 `voice-profile.md`：用本人会说的词，避开忌用词（标题尤其容易冒 AI 腔）。

## 10 种型号

| 型号 | 适合 | 骨架示例 |
|---|---|---|
| 数字清单 | 方法/经验，给确定性、扫读友好 | 「N 个 X」「3 步搞定 X」「关于 X 的 5 个误区」 |
| 痛点直击 | 引共鸣 | 「为什么你越 X 越 Y」「X 了这么久，还是 Y」 |
| 反常识 | 涨粉/立观点 | 「别再 X 了」「真正让我 Y 的不是 X」「90% 的人都搞错 X」 |
| 身份圈定 | 精准锁人群 | 「30 岁还在 X 的人」「做 X 的，看看这条」 |
| 结果前置 | 有成果可晒 | 「靠 X，我一个月 Y」「从 0 到 X，我只做对一件事」 |
| 好奇缺口 | 留悬念 | 「我做对了一件事」「X 这件事没人告诉你」 |
| 前后对比 | 有转变 | 「从 X 到 Y」「以前我 X，现在我 Y」 |
| 提问代入 | 引共鸣/评论 | 「你有没有也 X 过？」「为什么 X？」 |
| 干货承诺 | 教程/攻略 | 「一篇讲透 X」「X 保姆级教程」「X 全攻略」 |
| 情绪共鸣 | 情绪/故事 | 「成年人的 X」「X 的人，其实都很 Y」 |

## 出标题的动作
1. 看这条内容的**目标**（引共鸣→痛点/提问/情绪；涨粉→反常识/身份；卖货→结果前置/干货承诺）。
2. 选 2-3 个**不同型号**，各出一个，套上用户的词。
3. 列给用户挑，并问一句「哪个最像你会发的」——把用户的选择记进档案的「修辞&叙事偏好」。
