# 面试：一轮问完，给默认，能猜就不问

目标：用**最少的打断**问清四件事。用户多半讨厌挤牙膏式提问，喜欢你把话替用户补全。所以原则是：

> **能从碎片 + 声音档案里推断的，直接定下来并说一句「我先按 X 来，不对你说」；只有真的拿不准的才问。**

## 怎么问（用 AskUserQuestion 批量问）

把下面四个问题**放进一次 AskUserQuestion 调用**（一个 call 里 4 个 question），一屏问完，别分四轮。每个问题的**第一个选项放你的推荐**，并在 label 后标「（推荐）」，description 里写清为什么推荐。

如果某个维度你已经很确定（比如用户明说「发朋友圈」），那一题就别放进去，直接采用。最少可以只问 1 个问题。

---

## Q1 · 发哪个平台

| 选项 | 适合 |
|---|---|
| 朋友圈 | 短、私人、即兴；记录心情 / 一句感悟 / 顺手安利 |
| 推特 / X | 短而锐；一个观点 / 一条 thread / 行业碎念 |
| 小红书 | 标题党 + 正文 + 标签；知识付费引流主战场 |
| 公众号 | 长文；把一件事讲透 / 立深度人设 |
| 其它 | 即兴问用户：什么平台？字数限制？给谁看？ |

**怎么猜推荐：** 想法越短越情绪化 → 朋友圈/推特；越像「能帮到别人的方法/清单」→ 小红书；越像「一件需要展开论证的事」→ 公众号。**结合档案**：按用户档案里写明的「主平台/主战场」给默认，含糊时它的优先级最高。

## Q2 · 想达到什么（决定框架，不是决定语气）

| 选项 | 写法会怎样变 |
|---|---|
| 记录自己 | 真实 > 好看，可以不完整、不收尾，允许私人 |
| 引共鸣 | 找一个「很多人有但没说出口」的痛点开头 |
| 涨粉 / 立人设 | 给一个鲜明的观点或反常识，让人想关注「这个人」 |
| 卖货 / 转化 | 价值前置 + 一个明确的下一步动作，但忌硬广 |

**怎么猜推荐：** 看碎片里的情绪。抱怨/感慨 → 共鸣；方法/经验 → 涨粉或卖货；纯心情 → 记录。

## Q3 · 用什么结构（按平台给模板，详见 platforms.md）

不要给通用结构，给**所选平台**的 2-3 个具体模板让用户选。例如小红书：
- **痛点钩子式**：戳痛点 → 我也踩过 → 3 个解法 → 一句收口
- **故事复盘式**：一个具体场景 → 转折 → 我学到的
- **反常识清单式**：「关于 X，90% 的人都搞错了」→ 逐条打脸 → 正解

朋友圈/推特通常不需要复杂结构，可把 Q3 降级成「要不要带个钩子开头」。

## Q4 · 什么叙事角度

| 选项 | 一句话 |
|---|---|
| 故事 | 用一个具体场景/人物带出来，最不像 AI |
| 观点 | 亮明立场，正面论证 |
| 清单 | N 条，扫读友好（小红书常用） |
| 对话 | 自问自答 / 还原一段对话 |
| 反常识 | 先推翻一个共识，再给新解 |

**怎么猜推荐：** 档案里用户偏好哪种叙事就默认哪种（见 voice-profile「修辞 & 叙事偏好」）。没记录时：小红书默认清单或痛点钩子，公众号默认故事或观点，朋友圈/推特默认观点或反常识。

---

## 不问的东西（用档案默认，别浪费用户的注意力）

- **语气 / 人称 / emoji 量 / 标点习惯** → 一律走 `voice-profile.md`，用户不主动提就不问。
- **字数** → 走 `platforms.md` 的平台默认，除非用户指定。
- **要不要做成卡片** → 等文字定稿后再在 Step 6 问，别一开始问。

## 面试完，复述一遍再动笔

四题答完，用一句话把方案锁定给用户看，让用户有机会喊停：
> 「OK：发**小红书**，目标**引共鸣**，用**痛点钩子式**，**故事**角度，按你一贯的口语 + 少量 emoji。我写了。」
