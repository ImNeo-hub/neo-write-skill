# 声音引擎：建档 · 写作 · 学习

这是这个 skill「越用越像你」的发动机。三件事：**冷启动建档**（第一次用）、**写作时怎么用档案**、**每次用完怎么更新档案**。

档案唯一真源：`memory/voice-profile.md`。样本放 `memory/samples/`。

---

## A. 冷启动校准（档案为空时，先做这个）

写出来像不像用户本人，取决于你有没有读过用户真正写过、并且满意的东西。空档案直接写 = AI 腔。所以第一次必须建档：

**首选 · 喂样本（最准）**
1. 跟用户说：「想写得像你、不像 AI，我得先读几篇你自己写过、觉得'这就是我'的东西。给我 3–5 篇——朋友圈、推特、小红书、公众号都行，越口语越真实越好。」
2. 用户贴来 / 指给你文件路径（公众号导出、小红书旧帖、聊天记录、笔记都行）。
3. 把每篇原样存进 `memory/samples/`，文件名带平台，如 `xhs-01.md`、`pyq-03.md`。
4. 按下面「档案维度」抽特征，填进 `voice-profile.md`。
5. **把抽出来的档案念给用户听**，让用户纠正：「我其实从不用感叹号」「这个词我不会说」。纠正后再存。

**备选 · 没样本时，快问几题**
样本一时拿不出，就问 5–6 道快选题先建个 v1，之后边用边修：
- 你写东西偏**短句利落**还是**长句铺陈**？
- 你说话更像**跟朋友唠**还是**台上讲**？
- emoji：**爱用 / 少量 / 基本不用**？
- 你**会不会**带点脏话 / 网络梗 / 自嘲？
- 有没有**一看就烦、绝不想出现在你文字里**的词？（先建黑名单）
- 你习惯怎么**开头**、怎么**收尾**？

## B. 档案维度（`voice-profile.md` 里抽这些）

抽特征要**具体、可执行**，不要「语言生动」这种废话。要「爱用破折号把话接下去」「句号经常省」这种能直接照做的。

1. **身份与立场** — 谁在说话、用户被认作什么、反复出现的观点/偏见、用户挺什么反对什么。
2. **句子节奏** — 平均长短、长短句怎么混、用不用残句/碎句、会不会一逗到底。
3. **词汇偏好** — 爱用词/口头禅（列具体词）；口语还是书面；方言/网络词；**忌用词**单列。
4. **语气词 & 标点** — 吧/啊/呢/嘛/哈 用不用；破折号——、省略号……、句号、感叹号的习惯；用不用「」做强调。
5. **开头习惯** — 怎么起头（提问？甩结论？画面？自嘲？）。
6. **结尾习惯** — 怎么收（开放问题？金句？CTA？戛然而止？）。
7. **修辞 & 叙事偏好** — 比喻风格、爱讲具体例子还是抽象、反问/排比/自嘲、默认叙事角度。
8. **emoji 习惯** — 哪几个、放哪、密度（**分平台**记，差别很大）。
9. **平台分身** — 朋友圈/推特/小红书/公众号 各自的字数·正式度·emoji 差异。
10. **样本指针** — 指向 `samples/` 里最能代表用户的几篇。

---

## C. 写作时怎么用档案

1. 读 `voice-profile.md` 全文 + `samples/` 里**同平台** 1-2 篇当 few-shot。
2. 先按档案的开头习惯起头，按节奏铺句，落用户爱用的词，避开忌用词。
3. **交稿前默跑 AI 腔黑名单**（下面 D），命中就改。
4. 故意留一点「人味毛边」：一处口语、一个不那么对称的句子、一句用户会说的题外话。别追求完美——完美就是 AI 的指纹。

---

## D. AI 腔黑名单（交稿前自检，命中即改）

这些是「**疑似** AI 指纹」。但**最终权威是声音档案**——如果档案说用户真的爱用某个，就保留。判断标准永远是「用户本人会不会这么说」。

- **连接词病**：首先 / 其次 / 再次 / 最后 / 总而言之 / 综上所述 / 总的来说
- **万能开头**：在这个 ×× 的时代 / 随着 ×× 的发展 / 众所周知 / 不得不说 / 说到 ××
- **商业黑话**：赋能 / 抓手 / 闭环 / 对齐 / 颗粒度 / 底层逻辑（滥用）/ 打法 / 心智 / 势能 / 壁垒（滥用）
- **空心强调**：满满的干货 / 纯纯的 / 真的真的 / 绝绝子 / yyds（除非确属用户）
- **对偶病**：不仅……更是…… / 既要……又要…… / 不是……而是……（滥用）/ 与其……不如……
- **标点病**：每句都加感叹号；破折号 —— 满天飞；动不动用「」强调
- **列表病**：什么都列 1234；emoji 项目符号 ✅🔥💡 堆砌
- **结尾病**：希望对你有帮助 / 你学会了吗 / 快去试试吧 / 记得点赞收藏关注 / 我们下期再见
- **完美病**（最致命）：每段一样长；每个观点都配三个例子；全程不跑题、不卡顿、从不说「我也没太想清楚」

---

## E. 学习回路（每次定稿后必做，否则 skill 不会变聪明）

1. **抓用户的修改**：对比你的草稿 vs 用户改后的定稿。重点看：
   - 用户把哪个**词**换成了什么 →（例：把「赋能」改成「帮上忙」=「忌商业黑话，偏大白话」）
   - 用户**删**了什么 →（例：删掉了结尾的「快去试试吧」=「忌号召式收尾」）
   - 用户说了什么**元评价** →（「太端着了」「这不像我」「太满了」）
2. **要个信号**：顺口问「这条几分像你说的话？（1-5）」。≤3 就追问「哪里最不像？」
3. **写回档案**：把新规律追加到 `voice-profile.md` 末尾的「学习日志」，每条带日期，格式：
   `- [2026-06-07] 把"赋能"改成"帮上忙" → 忌商业黑话，偏大白话`
   特别好的定稿，整篇存进 `samples/` 当新范例。
4. **定期消化**：学习日志攒到 ~15 条，做一次合并——把反复出现的规律提炼进上面 1-10 的正式维度，删掉重复条目，保持档案精炼。
