# 平台规范：字数 · 结构 · 语气 · 接力

每个平台的「物理规则」不同。声音档案管**怎么说话**，这个文件管**在哪个平台该用什么形状**。冲突时：字数/结构听平台，语气/用词听声音档案。

> 接力一律**工具无关、可选**：有出图/长文/标题工具就接，没有就只交付文字。引擎不硬依赖任何外部 skill。

---

## 朋友圈

- **字数**：30–150 字最佳。超过 6 行会被折叠成「全文」，第一行就是钩子。
- **没有标题**，第一句即生死。
- **结构模板**
  - 一句感悟式：一个画面/一件小事 + 一句你的判断。（最高频）
  - 顺手安利式：我在用/在读的东西 + 一句为什么 + 不强求。
  - 留白提问式：抛一个问题或半句话，引评论区。
- **语气**：最私人、最松弛。可以有错别字感的真实，可以不收尾。
- **忌**：像广告、像小作文、配一堆话题标签。
- **emoji**：少量或没有，看档案。

## 推特 / X

- **字数**：单条 ≤ 140 中文字（约 280 字符）。要展开就拆 **thread**，每条能独立成立。
- **结构模板**
  - 单条观点弹：一句反共识/锐评，点到为止。
  - Thread 式：第 1 条是最强钩子（承诺一个收获）→ 中间每条一个点 → 末条收口或 CTA。
  - 金句 + 展开：先甩结论，再用 1-2 条解释。
- **语气**：锐、快、敢站队。短句优先，一条只说一件事。
- **忌**：中文「小红书腔」（满屏 emoji、「家人们」）；把长文硬塞进一条。
- **emoji / 标签**：克制；hashtag 最多 1-2 个，且要自然。

## 小红书

- **标题**：最高优先级，决定打开率。20 字内，带钩子（数字 / 痛点 / 反差 / 身份）。憋不出来去 `titles.md` 套公式。
- **正文**：300–800 字，**多用换行和空行**，扫读友好。开头 1-2 行要留住人（小红书会截断）。
- **结构模板**
  - 痛点钩子式：戳痛点 → 我也踩过 → 3 个解法 → 一句收口
  - 故事复盘式：一个具体场景 → 转折 → 我学到的 → 你也可以
  - 反常识清单式：「关于 X，90% 的人都搞错」→ 逐条打脸 → 正解
- **语气**：真诚 + 利用户，像跟朋友掏心窝，不端着。若用户做**知识付费 / 引流**，正文要给真东西，结尾自然带软钩子（「想要的扣 1 / 主页」），别硬广。
- **标签**：结尾 5-10 个 `#话题`，混「大词 + 精准词」。
- **emoji**：适量，分点处用，看档案别堆砌。
- **接力（可选）**：文字定稿 → 若有卡片/出图工具，可提议做成 3:4 图文；没有就给配图建议。

## 公众号

- **字数**：1500–4000 字长文，把一件事讲透。
- **标题**：双标题思维——一个给打开率，一个给老读者的真实感。去 `titles.md` 借公式。
- **结构模板**
  - 故事开篇式：一个具体故事/场景切入 → 抛出问题 → 逐层展开 → 落到一个观点
  - 观点论证式：亮明反共识结论 → 三段论据（各带例子/数据/故事）→ 升华
  - 复盘拆解式：一件你做过的事 → 拆成可复制的步骤/认知 → 给读者的迁移
- **语气**：可以慢、可以有「我也没完全想通」的真实毛边，这正是反 AI 的地方。段落长短交错，别每段一样长。
- **忌**：小标题用「一、二、三」干巴罗列；结尾「希望对你有帮助」「我们下期再见」。
- **接力（可选）**：要某种特定人设/风格的长文，且用户有对应工具，可提议接力；封面图同理交给出图工具。

## 其它平台（即兴处理）

用户给一个没列出的平台（知乎 / B 站文案 / 视频脚本 / 即刻 / 群发……）时，现场问三件事：
1. 字数 / 时长上限？
2. 给谁看（一个具体的人或人群）？
3. 看完想让对方做什么 / 记住什么？
然后套最接近的一档规范，语气仍走声音档案。
